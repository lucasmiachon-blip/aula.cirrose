# Narrative Arc — Cirrose Masterclass

> Mapa narrativo para coding agents. Cada slide tem um PROPÓSITO emocional.
> Sem este contexto, refatorar = destruir pacing.
> Fonte: Bíblia Narrativa (Notion — ver `docs/SYNC-NOTION-REPO.md`)

## Premissa

Masterclass segue o **Sparkline de Duarte**: alternância contínua entre
"o que é" (realidade clínica) e "o que poderia ser" (ação que muda desfecho).
O paciente-caso (Seu Antônio) é o fio condutor — sua trajetória clínica
acompanha os 3 atos, com o case panel evoluindo de neutro → perigo → esperança.

**Subtítulo programático:** *Classificar · Intervir · Reverter*

## Case Panel — Evolução

```
s-hook    → neutral   (apresentação)
s-cp1     → caution   (decisão necessária)
s-cp2     → danger    (paciente piorando)
s-cp3     → hope      (recompensação possível)
s-close   → resolved  (timeline completa)
```

Transições intermediárias herdam o último estado registrado.
Ex: s-a2-01 a s-a2-06 mantêm `caution` (herdado do CP1).

## Regras para Agentes

1. **Nunca revelar resposta antes do checkpoint.** Se o slide ensina albumina (s-a2-03), o CP2 DEPOIS é onde o paciente a recebe. Não antecipar no body.
2. **Respeitar gradiente de tensão.** Ato 1 = didático, calmo. Ato 2 = urgente, cada slide é ação. Ato 3 = alívio, esperança.
3. **Callbacks ao hook.** Qualquer refactor do s-close deve ecoar "5 números" e "3 decisões". Qualquer refactor do hook deve prometer exatamente isso.
4. **Chekhov's Guns ativos:**
   - "Caminhoneiro" → encefalopatia (s-a2-06) impede ele de trabalhar
   - "ATTIRE" → armadilha (s-a2-03) é setup, CP2 é payoff
   - "LSM 32→18" → recompensação (s-cp3) é o arco completo do Seu Antônio
5. **Cinematic blackouts** (planejados, não implementados): slides navy puro com 1 pergunta, zero UI. Candidatos: antes de cada checkpoint.
6. **Dados clínicos são sagrados.** Antes de refatorar qualquer slide com NNT, HR, ou p-value, executar `grep "s-{id}" references/evidence-db.md` e verificar PMID. Nunca arredondar, inventar, ou omitir IC 95%.
